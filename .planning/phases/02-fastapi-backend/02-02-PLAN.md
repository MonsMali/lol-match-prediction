**api/routers/predict.py**:
- Create `APIRouter(prefix="/api", tags=["prediction"])`.
- `POST /predict` endpoint. Declare as plain `def` (NOT `async def`) so FastAPI auto-threads the blocking ML inference. Accept `payload: PredictRequest` and `adapter: LoLDraftAdapter = Depends(get_adapter)`.
- Convert PredictRequest to a dict matching DraftInput structure:
  ```python
  draft_dict = {
      "blue_team": payload.blue_team,
      "red_team": payload.red_team,
      "blue_picks": payload.blue_picks.model_dump(),  # {top, jungle, mid, bot, support}
      "red_picks": payload.red_picks.model_dump(),
      "blue_bans": payload.blue_bans,
      "red_bans": payload.red_bans,
      "patch": payload.patch,
  }
  ```
- Validate champion names BEFORE calling adapter: collect all 20 champion names (10 picks + 10 bans), check each against `adapter.valid_champions`. For unknown champions, attempt normalization via `normalize_champion_name` from `src.adapter.normalization`. If still unknown after normalization, return 422 with `detail` listing the invalid champions. This is the "fail fast" behavior from CONTEXT.md.
- For team names: per CONTEXT.md decision, unknown teams are accepted with silent fallback. Wrap the `adapter.predict_from_draft(draft_dict)` call in a try/except for ValueError. If the error is about team names, catch it and retry with a fallback approach (or let the adapter handle it -- check if the adapter already falls back gracefully or raises). If the adapter raises ValueError on unknown teams, catch it at the API layer and substitute a known team name (e.g., pick the first team from `adapter.valid_teams`), then retry prediction. Log a warning but do not return an error to the user.
- On success, return `PredictResponse(blue_win_probability=result.blue_win_prob, red_win_probability=result.red_win_prob)`.
- On ValueError from adapter (other than team issues), return `HTTPException(status_code=422, detail=str(e))`.

**api/routers/champions.py**:
- Create `APIRouter(prefix="/api", tags=["champions"])`.
- `GET /champions` endpoint (async def). Accept `adapter: LoLDraftAdapter = Depends(get_adapter)`.
- Iterate over `sorted(adapter.valid_champions)`. For each champion name, call `get_ddragon_url(name)` from `api.champion_mapping`.
- Return `ChampionListResponse` with a list of `ChampionInfo(name=name, key=name, image_url=url)` for each champion. The `key` field is the canonical name used in the adapter (same as the display name from Oracle's Elixir format).
- Response is cacheable (champion list does not change during runtime). Add a comment noting frontend can cache this.

**api/routers/teams.py**:
- Create `APIRouter(prefix="/api", tags=["teams"])`.
- `GET /teams` endpoint (async def). Accept `adapter: LoLDraftAdapter = Depends(get_adapter)`.
- Start with `TEAMS_BY_LEAGUE` from `api.team_data`. Filter each league's team list against `adapter.valid_teams` to only include teams the model actually knows about.
- Collect all teams from `adapter.valid_teams` that are NOT in any league. Add them under an `"Other"` key.
- Return `TeamListResponse(teams=result_dict)`.
