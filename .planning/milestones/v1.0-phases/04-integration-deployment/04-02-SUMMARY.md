## Decisions Made
- Used raw `fetch('/health')` instead of the `apiFetch` wrapper for health polling, since WarmUpScreen must function before QueryClient is relevant and network errors should be silently retried.
- Used inline `<style>` tag for WarmUpScreen CSS rather than Tailwind classes to avoid any class-loading race condition during the cold-start loading screen.
- Set `autoDeploy: false` in render.yaml to prevent accidental deployments from every push to the repository.

## User Setup Required

External services require manual configuration for production deployment:

**Render (hosting):**
1. Create a new Web Service on https://dashboard.render.com connected to the GitHub repo (or use the render.yaml Blueprint)
2. Optionally set a custom MODEL_UPLOAD_TOKEN environment variable in the Render dashboard (one is auto-generated by render.yaml)

**cron-job.org (keep-alive):**
1. Create a free account at https://cron-job.org
2. Add a cron job that sends GET to `https://<your-render-url>/health` every 10 minutes to reduce free-tier spin-down
