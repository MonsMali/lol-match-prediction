**src/adapter/validation.py:**

Create a `validate_draft(draft: DraftInput, valid_champions: set[str], valid_teams: set[str], aliases: dict[str, str]) -> DraftInput` function that:

1. Normalizes and validates both team names using `normalize_team_name` from normalization.py
2. Normalizes and validates all 10 picks (blue_picks values + red_picks values) using `normalize_champion_name`
3. Normalizes and validates all 10 bans using `normalize_champion_name`
4. Validates role keys in blue_picks and red_picks: must be exactly {"top", "jungle", "mid", "bot", "support"} -- raise ValueError if missing or extra roles
5. Validates blue_bans and red_bans are each exactly 5 items -- raise ValueError if not
6. Checks for duplicate champions across ALL picks and bans (20 champions total must all be unique) -- raise ValueError listing the duplicates
7. Returns a NEW DraftInput with all names normalized to their canonical form

All validation errors should be `ValueError` with descriptive messages. Validate in order so user gets the first meaningful error.

**src/adapter/features.py:**

Create `compute_features_for_side(...)` that computes the 37-feature vector for one side of the match. This is the critical function -- it must replicate the exact feature computation from `create_advanced_features_vectorized` + `apply_prediction_time_encoding` but for a single match without any DataFrame manipulation.

Parameters:
- `picks: dict[str, str]` -- {role: champion_name} (already normalized)
- `bans: list[str]` -- 5 ban champion names (already normalized)
- `team: str` -- team name (already normalized)
- `side: str` -- "Blue" or "Red"
- `patch: str` -- patch string
- `year: int` -- extracted from patch or provided
- `league: str` -- team's league
- `playoffs: int` -- 0 or 1
- `split: str` -- split name
- `champion_chars: dict` -- champion_characteristics lookup
- `meta_strength: dict` -- champion_meta_strength lookup (keyed by (patch, champion) tuples)
- `champion_pop: dict` -- champion_popularity lookup
- `team_perf: dict` -- team_historical_performance lookup
- `ban_priority: dict` -- ban_priority lookup
- `encoders: dict` -- target encoders for categorical features

Returns: `list[float]` -- exactly 37 values in the EXACT order matching the expected_features list from RESEARCH.md.

Implementation guidance -- reference RESEARCH.md's feature list and the existing source code:

1. **Champion characteristics (features 0-5):** Average win_rate, early_game_strength, late_game_strength across 5 picks. Compute scaling (avg late - avg early), composition_balance (std of roles), avg flexibility. Use `champion_chars.get(champ, default_char)` for each.

2. **Team-aggregated champion stats (features 6-9):** team_avg_winrate, team_early_strength, team_late_strength, team_flexibility -- these may be similar to group 1 but computed differently in the original code. CHECK the original `create_advanced_features_vectorized` carefully.

3. **Meta features (features 10-13):** team_meta_strength (avg meta strength for picks), team_meta_consistency (std), team_popularity (avg), meta_advantage (meta_strength - 0.5). Use `meta_strength.get((patch, champ), 0.5)` and `champion_pop.get((patch, champ), {'popularity': 0.0})`.

4. **Ban features (features 14-16):** ban_count (always 5), ban_diversity (unique bans / 5), high_priority_bans (count of bans in ban_priority with above-median priority).

5. **Team performance (features 17-20):** From `team_perf.get(team, default_perf)`: overall_winrate, recent_winrate, form_trend, games_played (as experience).

6. **Match context (features 21-23):** playoffs (0/1), side_blue (1 if Blue else 0), year.

7. **champion_count (feature 24):** Always 5 (number of unique champions picked).

8. **Interaction features (features 25-27):** meta_form_interaction, scaling_experience_interaction, composition_historical_winrate. Compute as products of the component features.

9. **Target-encoded categoricals (features 28-36):** league, team, patch, split, and 5 role champions. Use `encoders.get(col, {}).get(value, default)` where default is the global mean from the encoder.

CRITICAL: The feature ORDER must match exactly. Hard-code the expected_features list and construct the output vector in that order. If the order is wrong, predictions will be silently incorrect.

CRITICAL: Do NOT use pandas DataFrames. Compute everything with plain Python dicts and numpy for math operations only (mean, std).

CRITICAL: Reference `src/feature_engineering/advanced_feature_engineering.py` methods `create_advanced_features_vectorized` and the predictor's `apply_prediction_time_encoding` to ensure exact replication. If any feature computation is ambiguous, match the existing code exactly.

**src/adapter/adapter.py:**

Create `LoLDraftAdapter` class implementing the singleton pattern per CONTEXT.md decisions:

```python
class LoLDraftAdapter:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def __init__(self, artifacts_dir: Path = None):
        if hasattr(self, '_initialized'):
            return
        # ... eager loading ...
        self._initialized = True
```

**__init__ (eager loading, fail-fast):**
1. Resolve `artifacts_dir` -- default to `MODELS_DIR` from `src.config`. Use the audit findings from Plan 01 SUMMARY to determine exact file paths.
2. Load all required artifacts using `joblib.load`:
   - Model file (path determined by Plan 01 audit -- likely `ultimate_best_model.joblib` or extracted from dict)
   - Scaler file (likely `ultimate_scaler.joblib`)
   - Feature engineering object (`ultimate_feature_engineering.joblib`) -- extract lookup dicts from it
   - Encoders (from the audit findings)
3. Extract lookup dicts from feature engineering object: `champion_characteristics`, `champion_meta_strength`, `champion_popularity`, `team_historical_performance`, `ban_priority`
4. Build `valid_champions` set from `champion_characteristics.keys()`
5. Build `valid_teams` set from `team_historical_performance.keys()`
6. Determine `latest_patch` from `max(set(patch for patch, _ in self.meta_strength.keys()))` per RESEARCH.md advice
7. Store `model_name` (class name of the model) and `model_version` (from `model._sklearn_version` or sklearn version)
8. If ANY artifact file is missing, raise `FileNotFoundError` with a clear message listing exactly which file(s) are missing and their expected paths. Do NOT catch and continue.

**predict_from_draft(self, draft: DraftInput) -> PredictionResult:**
1. Call `validate_draft(draft, self.valid_champions, self.valid_teams, CHAMPION_ALIASES)` -- returns normalized draft
2. Determine patch: use `draft.patch` if provided, else `self.latest_patch`
3. Determine additional context: year (extract from patch or use latest year), league (look up from team data or default), playoffs (default 0), split (default to latest)
4. Compute features for BLUE perspective: `compute_features_for_side(blue_picks, blue_bans, blue_team, "Blue", patch, ...)`
5. Compute features for RED perspective: `compute_features_for_side(red_picks, red_bans, red_team, "Red", patch, ...)`
6. Scale both feature vectors using the loaded scaler
7. Get predictions: `blue_pred = model.predict_proba(blue_features)` and `red_pred = model.predict_proba(red_features)`
8. Average using dual-perspective formula: `avg_blue_win = (blue_pred[0][1] + (1 - red_pred[0][1])) / 2`
9. Return `PredictionResult(blue_win_prob=avg_blue_win, red_win_prob=1-avg_blue_win, model_name=self.model_name, model_version=self.model_version)`

CRITICAL: The dual-perspective averaging formula MUST match the existing code exactly. Check `predict_match` in `interactive_match_predictor.py` to verify the indexing.

CRITICAL: The scaler expects a 2D array. Ensure feature vectors are reshaped: `np.array(features).reshape(1, -1)`.

CRITICAL: No `pd.read_csv` anywhere in this file. No DataFrame of the training data. All feature computation uses the lookup dicts.

**get_status(self) -> AdapterStatus:**
1. Collect list of loaded artifact names
2. Get memory usage via `resource.getrusage(resource.RUSAGE_SELF).ru_maxrss / 1024` (Linux, in MB) or `psutil.Process().memory_info().rss / (1024*1024)` if psutil available
3. Return `AdapterStatus` with all fields populated
