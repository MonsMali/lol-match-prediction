### E2E Flow: Page Load to Prediction

All 11 steps connected end-to-end:

1. Browser fetches `/` -> FastAPI SPA catch-all serves `index.html`
2. React mounts -> `WarmUpScreen` polls `GET /health` every 2s
3. Model ready -> children render (DraftBoard, ChampionGrid)
4. Champion grid loads via `GET /api/champions` with DDragon URLs
5. Team selection via `GET /api/teams` grouped by league
6. Draft entry via Zustand store with 20-step professional sequence
7. Role assignment after 5 picks per side
8. Predict button enables when `isDraftReady()` returns true
9. `POST /api/predict` with role-keyed draft payload
10. Adapter returns `PredictionResult` with blue/red probabilities
11. `WinProbability` split bar renders proportional widths

**Full E2E path: CONNECTED at all steps.**

### Integration Issues

**Issue 1 (Medium): Env var name mismatch -- API-07**
- `api/dependencies.py` line 42 reads `os.environ.get("ADMIN_TOKEN")`
- `render.yaml` line 11 provisions `key: MODEL_UPLOAD_TOKEN`
- Impact: Admin upload returns 401 on deployed Render even with correct token
- Fix: One-line change in either file to align the env var name

**Issue 2 (Low): Dead isPredicting state -- DRAFT-06**
- `DraftBoard.tsx` line 12: `isPredicting` state declared, `setIsPredicting(true)` never called
- `WinProbability` spinner prop permanently `false`
- User still gets pending feedback from button text ("Predicting...")
- Fix: Remove dead state or wire to `prediction.isPending`
